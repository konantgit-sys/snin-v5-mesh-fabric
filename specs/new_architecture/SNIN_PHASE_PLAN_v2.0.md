## 0. ТРИЗ-АНАЛИЗ ПРОТИВОРЕЧИЯ

### Техническое противоречие (ТП)

**Улучшаемый А:** Функциональность (16 слоёв, 10 прикладных use cases)  
**Ухудшаемый Б:** Время реализации (95 дней → ∞ при попытке сделать всё сразу)

**Формулировка ТП-1:**  
ЕСЛИ реализовать все 16 слоёв последовательно,  
ТО архитектура будет полной,  
НО пользователи увидят результат через 2+ года.

**Формулировка ТП-2:**  
ЕСЛИ запустить только текущие 5 слоёв,  
ТО продукт будет доступен сейчас,  
НО архитектура останется неполной, проигрывая конкурентам.

### Идеальный конечный результат (ИКР)

**ИКР-1:** Каждая фаза даёт самостоятельную ценность — можно остановиться после любой фазы и иметь работающий продукт.  
**ИКР-2:** Фазы идут от максимальной ценности при минимальных затратах.  
**ИКР-3:** Каждая фаза не требует переписывания предыдущей.

## ФАЗА 1 — ИДЕНТИЧНОСТЬ И РЕПУТАЦИЯ (5-7 дней)
**Девиз:** «У каждого агента есть имя и вес»  
**Arch слои:** L5 Identity & Reputation  
**Нового кода:** ~300 строк (DID resolver + reputation score)

### Задачи:
```
1.1 DID resolver — did:snin:pubkey
    Каждый агент получает DID
    DHT хранит mapping: did → pubkey → capabilities
    
1.2 Reputation score
    История: сколько сообщений доставлено, ошибок
    Вес в gossip: reputation влияет на приоритет доставки
    
1.3 Soulbound attestations
    Кто и когда аттестовал агента
    Хранится в DHT K=3
```
**Done when:** У каждого агента DID, reputation влияет на маршрутизацию  
**ТРИЗ:** Принцип №5 — объединяем DID + репутацию = доверие без блокчейна

## ФАЗА 2 — ПРИВАТНОСТЬ И ZK (7-10 дней)
**Девиз:** «Докажи что знаешь, не раскрывая»  
**Arch слои:** L3.5 ZK Layer + L4.5 Privacy Mixnet  
**Нового кода:** ~500 строк (Merkle gate + onion routing)

### Задачи:
```
2.1 ZK для всех слоёв
    Merkle proof включения в DHT
    ZK proof что платёж валиден (выделить из zk_prover.py)
    ZK proof что агент — владелец DID
    
2.2 Onion routing (3-hop)
    Сообщение проходит через 3 случайных relay
    Каждый relay видит только следующий hop
    Аналог Tor, но легче (нет цепей, только одноразовые)
    
2.3 Dandelion spreading
    Источник gossip скрыт
    Первые 2-hop — «стебель» (один peer)
    Потом «цветок» (broadcast)
```
**Done when:** Агент может отправить ZK-подписанное сообщение анонимно  
**ТРИЗ:** Принцип №2 (выделение) — ZK отделён от платежей, работает для всех слоёв
